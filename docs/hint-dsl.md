# Hint system — ідеї та концепти

Базова ідея: підказки у картках зберігаються як HTML прямо в `cards.yaml`.
Anki рендерить їх через шаблон + CSS. Ніякого парсингу — просто HTML-теги з класами.

---

## Варіанти підказок

### 1. Синоніми / альтернативи

Показати кілька українських варіантів, щоб зрозуміти яке англійське слово шукати.

```html
Він <span class="h-syn">(обґрунтував / виправдав)</span> свій вибір
```

Рендер: сірий текст, одразу зрозуміло що це "уточнення", не основний текст.

---

### 2. Граматична мітка

Маленький бейдж що показує який час / форма тренується.

```html
Він обґрунтував свій вибір <span class="h-gram">Past Simple</span>
```

Рендер: маленька кольорова таблетка (синя / фіолетова).

---

### 3. Нотатка про реєстр / нюанс

Коли важливо знати: це формально / розмовно / технічний жаргон.

```html
Він <span class="h-note">формально</span> обґрунтував рішення
```

Рендер: маленький курсив, сірий.

---

### 4. Ключове слово

Виділити саме те слово, яке тренується на цій картці.

```html
Він <span class="h-key">обґрунтував</span> свій вибір
```

Рендер: жирний + легкий жовтий підсвіт. Одразу фокус на потрібне слово.

---

## Круті UI-варіанти (ідеї)

### 5. Тултіп при наведенні

Українське слово виглядає чисто — але при наведенні / довгому тапі на мобільному
з'являється підказка. Підходить для складних нюансів.

```html
Він <span class="tip" title="justify · substantiate · back up">обґрунтував</span> свій вибір
```

Або кастомний тултіп:

```html
<span class="tooltip">
  обґрунтував
  <span class="tooltip-text">формально: justify<br>нейтрально: back up</span>
</span>
```

Рендер: слово з маленькою крапочкою знизу (пунктирний border-bottom). При наведенні — спливаюча підказка.

**Коли корисно:** коли не хочеш показувати підказку одразу — вона "захована", але доступна.

---

### 6. Розкривачка (spoiler / reveal)

Підказка схована, можна натиснути і побачити. Підходить для граматичних пояснень
або розширеного контексту.

```html
<details class="h-details">
  <summary>💡</summary>
  Цей час (Past Simple) — завершена дія в конкретний момент.
  Не плутати з Present Perfect.
</details>
```

Або у вигляді "blur spoiler" — текст розмитий, клік прибирає blur:

```html
<span class="spoiler">justified / substantiated</span>
```

CSS: `filter: blur(5px); cursor: pointer;` → при кліку клас `revealed` прибирає blur.

**Коли корисно:** граматичні пояснення, довгий контекст, якщо не хочеш підказки забивати простір.

---

### 7. Реєстр-бейдж

Окремий бейдж для стилю мовлення — як кольорова мітка.

```html
<span class="badge formal">formal</span>
<span class="badge informal">informal</span>
<span class="badge tech">tech</span>
<span class="badge slang">slang</span>
```

Різні кольори для кожного типу:
- `formal` → синій
- `informal` → оранжевий
- `tech` → зелений
- `slang` → сірий

---

### 8. Комбінована підказка

Кілька типів на одній картці — кожен по-своєму:

```html
Він <span class="h-key">обґрунтував</span>
    <span class="h-syn">(виправдав / підтримав)</span>
    своє рішення
    <span class="badge formal">formal</span>
    <span class="h-gram">Past Simple</span>
```

---

### 9. Підказка "тільки якщо помилився"

Ідея: підказки показуються тільки на Back (після відповіді), не на Front.
Тоді Front чистий → повне занурення. Back показує: що ти мав сказати + нюанси.

```html
<!-- front.html — чистий, без підказок -->
{{Front}}

<!-- back.html — після відповіді показуємо все -->
{{Front}}
<div class="hint-block">
  <span class="h-gram">Past Simple</span>
  <span class="h-note">formal register</span>
</div>
<hr>
{{Back}}
```

---

## Відкриті питання

- Де зберігати підказки: в існуючих полях (`Front`) чи в окремому полі `Hints`?
  - В `Front` — просто, але мішає текст і метадані
  - В `Hints` — чисто, але потребує нове поле в _meta.yaml + зміна шаблону
- Які підказки показувати на Front, які тільки на Back?
- Мобільний Anki: тултіпи не працюють на тач — треба альтернативу (tap to reveal?)
- Чи варто нормалізувати вже існуючі картки, чи тільки нові писати за новим стандартом?
